# 用户手册

**项目名称**：SE-ROS-Proj — Nav2 室内机器人多模块控制系统  
**适用版本**：ROS 2 Humble  
**日期**：2026-06-18

---

## 1. 简介

本系统提供：

- **仿真导航**：Gazebo 中差分驱动机器人 + SLAM + Nav2
- **任务管理**：JSON waypoint 任务，Action `/mission/run`
- **语义导航**：禁行区/软代价/偏好车道（可选）
- **安全监控**：传感器/TF/堵塞检测，联动任务暂停
- **批量测试**：多场景自动生成与顺序执行

---

## 2. 环境准备

### 2.1 依赖

- Ubuntu 22.04
- ROS 2 Humble
- Ignition Gazebo 6（`ros_ign_gazebo`）
- Nav2、SLAM Toolbox（通过 apt/rosdep）

### 2.2 构建

```bash
cd /home/liyicheng/SE-ROS-Proj
source /opt/ros/humble/setup.bash
rosdep install --from-paths src --ignore-src -r -y
colcon build --symlink-install
source install/setup.bash
export PATH="/usr/bin:/opt/ros/humble/bin:$PATH"
```

## 3. 快速开始 — 手动 Demo

### 3.1 生成演示房间（可选）

```bash
python3 src/sam_bot_nav2_gz/scripts/generate_scattered_room.py \
  --seed 42 --box-count 12
```

输出：`src/sam_bot_nav2_gz/world/demo_pillar_room.sdf`

### 3.2 启动全栈（推荐）

```bash
source /opt/ros/humble/setup.bash
source install/setup.bash

ros2 launch course_bringup mission_demo.launch.py \
  world_file:=demo_pillar_room.sdf \
  enable_semantic:=True \
  enable_safety:=True
```

**界面**：

- Gazebo：仿真环境与机器人
- RViz：地图、激光、`/plan` **绿色规划路径**、机器人模型

### 3.3 发送任务（若未自动运行）

新终端：

```bash
source install/setup.bash
ros2 action send_goal /mission/run course_interfaces/action/RunMission \
  "{mission_file: '/path/to/mission.json', max_retry_per_waypoint: 1, allow_skip_waypoint: false}"
```

示例 mission：`src/course_bringup/config/sample_mission.json`

## 4. 批量场景测试

### 4.1 一键 Batch（README 默认）

```bash
source /opt/ros/humble/setup.bash
source install/setup.bash
export PATH="/usr/bin:/opt/ros/humble/bin:$PATH"

GENERATE_SCATTERED_WORLD=True SCATTER_SEED=53 SCATTER_BOX_COUNT=7 \
ENABLE_SEMANTIC=True ENABLE_SAFETY=True RUN_HEADLESS=False \
RESULTS_DIR=/tmp/nav2_results_gui \
bash src/nav2_scenario_runner/scripts/run_batch.sh
```

默认 profile 为 `default_batch.yaml`：**2 个 pillar corridor case**（`pillar_round_001/002`），开启语义区域，**不含动态拥堵**。

### 4.2 四场景全量 Batch

```bash
PROFILE=src/nav2_scenario_runner/config/batch_profiles/full_batch.yaml \
ENABLE_SEMANTIC=True ENABLE_SAFETY=True \
bash src/nav2_scenario_runner/scripts/run_batch.sh
```

包含 corridor / room_inspection / **congestion** / fault_injection 四类场景。

### 4.3 环境变量

| 变量 | 默认 | 说明 |
|------|------|------|
| `PROFILE` | `default_batch.yaml` | batch case 矩阵 |
| `CASES_DIR` | `/tmp/nav2_cases` | 生成 case 目录 |
| `RESULTS_DIR` | `/tmp/nav2_results` | 日志与结果 |
| `TIMEOUT_SEC` | `900` | 单 case 超时（秒） |
| `RUN_HEADLESS` | `False` | `True` 无 GUI |
| `ENABLE_SAFETY` | `True` | 启动 safety_monitor |
| `ENABLE_SEMANTIC` | `True` | 使用 semantic overlay |
| `GENERATE_SCATTERED_WORLD` | `False` | 为 `True` 时先生成 pillar room |

### 4.4 分步执行

```bash
# 1. 生成 cases
python3 src/nav2_scenario_runner/scripts/generate_cases.py \
  --profile src/nav2_scenario_runner/config/batch_profiles/default_batch.yaml \
  --output-dir /tmp/nav2_cases

# 2. 运行 batch
python3 src/nav2_scenario_runner/scripts/run_batch.py \
  --cases-dir /tmp/nav2_cases \
  --results-dir /tmp/nav2_results \
  --launch-package nav2_scenario_runner \
  --launch-file run_single_case.launch.py \
  --timeout-sec 900 \
  --tee-output \
  --extra-launch-arg run_headless:=False \
  --extra-launch-arg enable_safety:=True \
  --extra-launch-arg auto_start_delay_sec:=75 \
  --extra-launch-arg nav2_startup_delay_sec:=50

# 3. 汇总
python3 src/nav2_scenario_runner/scripts/summarize_results.py \
  --results-dir /tmp/nav2_results
```

**结果文件**：

- `/tmp/nav2_results/results.csv`
- `/tmp/nav2_results/summary.json`
- `/tmp/nav2_results/{case_id}.log`

### 4.5 单 Case 调试

```bash
ros2 launch nav2_scenario_runner run_single_case.launch.py \
  scenario_file:=/tmp/nav2_cases/pillar_round_001_scenario.yaml \
  run_headless:=False \
  enable_safety:=True \
  auto_start_delay_sec:=75
```

## 5. 安全模块使用

### 5.1 查看安全状态

```bash
ros2 topic echo /safety/state
```

`level`：`0=SAFE`, `1=SLOW_DOWN`, `2=STOP_NOW`

### 5.2 手动触发

```bash
ros2 service call /safety/trigger_pause course_interfaces/srv/TriggerPause "{reason: 'manual test'}"
ros2 service call /safety/trigger_emergency_stop course_interfaces/srv/TriggerEmergencyStop "{reason: 'estop test'}"
ros2 service call /safety/request_recovery course_interfaces/srv/RequestRecovery "{strategy: 'replan', reason: 'test'}"
```

### 5.3 参数文件

`src/sam_bot_nav2_gz/config/safety_monitor.yaml`

关键参数：

- `blockage_ignore_during_spin: true` — spin 脱困时不误判堵塞
- `blockage_window_sec: 6.0`
- `sensor_timeout_sec: 10.0`

## 8. 常见问题

### Q1：一直刷 `WAITING_FOR_RESULT`

mission 正在等 Nav2 完成当前航点。若长时间不结束，检查 RViz 中 `/plan` 与 `/map` 是否覆盖目标点，可适当增大 `auto_start_delay_sec`。

### Q2：RViz 无绿线

确认 Nav2 已 active，Fixed Frame 为 `map`，添加 Path 显示 `/plan`。

### Q3：Gazebo 黑屏 / DISPLAY 错误

无显示器时使用 headless：

```bash
RUN_HEADLESS=True bash src/nav2_scenario_runner/scripts/run_batch.sh
```

### Q4：Recovery 旋转角度

Recovery BT 配置为先后退再 **180°** spin，见 `sam_bot_nav2_gz/config/behavior_trees/`。
