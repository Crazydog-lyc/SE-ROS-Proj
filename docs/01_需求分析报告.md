# 需求分析报告

**项目名称**：SE-ROS-Proj — Nav2 室内机器人多模块控制系统  
**版本**：main  
**日期**：2026-06-18

---

## 1. 项目背景与目标

本项目为软件工程综合实验课程的四人协作 ROS 2 工作空间，在官方 **sam_bot + Nav2 + Gazebo** 基线之上，扩展四类自研能力：

1. **任务状态机与执行**（FR-B）
2. **场景生成与批量自动化测试**（FR-A）
3. **语义代价地图**（FR-C）
4. **安全监控与联锁**（FR-D）

**总体目标**：实现可复现的室内导航 Demo，支持语义约束、安全暂停/恢复，以及多场景批量回归测试。

## 3. 功能需求

### 3.1 FR-A — 场景生成与批量测试（陆华均）

| 编号 | 需求描述 | 优先级 | 验收标准 |
|------|----------|--------|----------|
| FR-A-01 | pluginlib 注册多种 ScenarioGenerator | 高 | Corridor / RoomInspection / Congestion / FaultInjection 四类插件可加载 |
| FR-A-02 | 固定 seed 生成可复现场景 YAML | 高 | 同 seed 输出一致 waypoint 与 semantic 区域 |
| FR-A-03 | 每个 case 输出 mission.json + semantic_overlay | 高 | `generate_cases.py` 生成完整产物目录 |
| FR-A-04 | 批量流水线 generate → run → summarize | 高 | `run_batch.sh` 产出 `results.csv`、`summary.json` |
| FR-A-05 | 单 case launch 与 full_stack 集成 | 高 | `run_single_case.launch.py` 可独立运行 |
| FR-A-06 | 程序化 Gazebo 房间生成 | 中 | `generate_scattered_room.py` 输出 `demo_pillar_room.sdf` |

### 3.2 FR-B — 任务状态机（徐梓鸣）

| 编号 | 需求描述 | 优先级 | 验收标准 |
|------|----------|--------|----------|
| FR-B-01 | 定义 RunMission Action 与 MissionState 消息 | 高 | `course_interfaces` 编译通过 |
| FR-B-02 | `/mission/run` Action Server 驱动 waypoint 序列 | 高 | JSON mission 可顺序执行 |
| FR-B-03 | 显式状态机处理 Nav2/安全/超时事件 | 高 | pytest 状态转移用例 PASS |
| FR-B-04 | goal 超时与 Nav2 结果映射 | 高 | FAILED/CANCELED 进入重试或失败终态 |
| FR-B-05 | 订阅 `/safety/state` 暂停/恢复/取消 | 高 | STOP_NOW 触发 SAFETY_STOP 链 |
| FR-B-06 | 发布 `/mission/state` 进度 | 中 | RViz/脚本可订阅 |
| FR-B-07 | auto_run_mission 等待 Nav2 ready 后自动发 goal | 中 | batch 场景无需手动 action send |

### 3.3 FR-C — 语义代价地图（李熠城）

| 编号 | 需求描述 | 优先级 | 验收标准 |
|------|----------|--------|----------|
| FR-C-01 | 几何工具：圆/多边形/旋转矩形点包含、折线距离 | 高 | gtest 用例 PASS |
| FR-C-02 | SemanticZoneLayer：keepout / 软代价区 | 高 | global costmap 可见禁行/高代价 |
| FR-C-03 | 代价融合策略 max/add/overwrite | 高 | 参数可配置 |
| FR-C-04 | PreferredLaneLayer：偏好车道惩罚 | 中 | 编译启用，默认配置可 disabled |
| FR-C-05 | DynamicCongestionLayer：动态拥堵圆 | 中 | 订阅拥堵事件，TTL 生命周期 |
| FR-C-06 | nav2_params_semantic.yaml 集成 Nav2 | 高 | `enable_semantic:=True` 可切换 |

### 3.4 FR-D — 安全监控（苏易）

| 编号 | 需求描述 | 优先级 | 验收标准 |
|------|----------|--------|----------|
| FR-D-01 | 激光 scan 超时检测 | 高 | 超时 → PAUSED |
| FR-D-02 | TF 超时检测 | 高 | 超时 → EMERGENCY_STOP |
| FR-D-03 | 可选前方障碍距离 pause | 中 | 参数 `enable_obstacle_monitor` |
| FR-D-04 | 局部堵塞检测（cmd_vel vs odom） | 高 | 触发 recovery + cancel Nav2 |
| FR-D-05 | spin 脱困期间跳过堵塞误判 | 高 | `blockage_ignore_during_spin` |
| FR-D-06 | SafetyAwareNavigator 包装 BasicNavigator | 中 | Demo 脚本可复现 recovery |
| FR-D-07 | 安全服务 TriggerPause/EStop/Cancel/Recovery | 高 | launch test 可调用 |

---

## 5. 约束与假设

1. 定位采用 **SLAM Toolbox 在线建图**，batch 默认使用 `demo_pillar_room.sdf` 物理世界。
2. 场景 YAML 中的 semantic 区域写入 costmap overlay，与 Gazebo 柱障房间对齐。
3. Nav2 行为树 recovery 已定制：先后退 0.25m，再 spin **180°**（见 `behavior_trees/`）。
4. 四人模块通过 `course_interfaces` 解耦，联调顺序：mission↔safety → semantic → batch。
