## 三、注释规范

### 文件头（AI 生成部分）

```python
# ========================================================================
# 文件: nav2_mission_manager/mission_action_server.py
# 负责人: 徐梓鸣 | 需求: FR-B | PPT: 第15-16页 任务管理
# ========================================================================
#
# 【AI-PROMPT】
# 我在 ROS2 Humble + Nav2 课程项目里，需要 MissionActionServerNode。
# 请搭 Action Server 骨架：/mission/run、/mission/state、/safety/state；
# _execute_callback 主循环框架即可，状态机我自己写。
#
# 【AI-SCOPE】import · declare · register · 空壳类
# ========================================================================
```

C++ 用 `//`，SDF 用 `<!-- -->`，`.msg/.srv/.action` 用 `#`。

### 手写部分

```python
# TODO[徐梓鸣]：FR-B-02 实现 /mission/run Action Server，驱动 waypoint 序列
class MissionActionServerNode(Node):
    def __init__(self, ...):
        # TODO[徐梓鸣]：FR-B-06 周期性发布 /mission/state
        ...
```

- 格式：`TODO[姓名]：FR-xx-yy 需求说明`
- 放在 class / 函数**上方**或函数体**首行**（缩进对齐）
- 联调可加：`# LINK[苏易]：/safety/state STOP_NOW`

### 不要写

- ~~【HAND-SCOPE】~~
- ~~「以下为业务实现（手写）」~~
- ~~把 TODO 插在 `class` 和 `def __init__` 之间（不缩进）~~

## 四、Cursor 提示词写法

1. 先说背景：`我在 ROS2 Humble + Nav2 课程项目…`
2. 说清要什么：类名、话题、接口
3. 说清不要什么：`状态机/融合逻辑我自己写`
4. PPT 可展示 2–3 轮迭代：骨架 → 补测试 → 联调
